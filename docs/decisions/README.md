# Architecture Decision Records

This directory contains ADRs for architecturally significant decisions.

Format: `ADR-<NNNN>-<short-kebab-title>.md`

Each ADR follows the standard template:
- **Context** — why this decision is needed
- **Decision** — what was decided
- **Consequences** — implications and trade-offs
- **Related** — links to specs, features, or other ADRs
